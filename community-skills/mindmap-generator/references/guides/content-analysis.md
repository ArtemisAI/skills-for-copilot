# 内容分析指南

本文档提供详细的内容分析流程，用于从各种输入源提取和准备思维导图内容。

## 文本描述分析

### 识别主题

从用户输入中提取核心主题：
- 关键词提取（如"微服务架构"、"React Hooks"）
- 主题类型识别（概念、技术、流程、知识体系）

### 提取要点

分析文本结构，识别：
- 核心概念和定义
- 关键特征或组成部分
- 层次关系和分类维度
- 应用场景和示例

## 文件内容分析

### Markdown 文件

- 保留现有标题层级作为参考
- 提取关键段落和列表
- 识别代码示例和引用

### 代码文件

- 分析文件结构和目录组织
- 提取关键类、函数和模块
- 识别架构模式和设计思路

### 文档文件

- 提取目录和章节结构
- 识别核心概念和术语
- 保留重要定义和说明

## URL 内容分析

使用 SearXNG 获取网页内容后：

1. **标题和大纲提取**
   - 提取 h1-h6 标题
   - 识别文章结构

2. **关键段落识别**
   - 摘要和引言
   - 核心论点和结论
   - 代码示例和图表说明

3. **元数据提取**
   - 作者和来源
   - 发布日期（判断时效性）
   - 相关链接

## 代码仓库分析

分析仓库结构：

1. **目录结构**
   - src/, lib/, tests/ 等主要目录
   - 配置文件和文档

2. **关键文件**
   - README, CONTRIBUTING, ARCHITECTURE
   - package.json, requirements.txt 等配置

3. **代码组织**
   - 模块划分
   - 依赖关系
   - 设计模式

## 内容质量评估

分析完成后，评估内容：

- **完整性**: 是否涵盖主题的主要方面
- **深度**: 内容是否足够详细
- **结构**: 是否有清晰的层次关系
- **准确性**: 信息是否准确可靠

如果内容不足，考虑：
- 使用 SearXNG 补充查询相关资料
- 提示用户提供更多信息
- 使用模板填充标准结构
